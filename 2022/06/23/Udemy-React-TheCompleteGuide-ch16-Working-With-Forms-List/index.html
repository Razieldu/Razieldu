What’s So Complex About Forms?

forms and their input can assume different states

主要有兩種狀態

  1. one or more input are invalid

=> output input-specific error messages & highlight problematic inputs

=> ensure form can not be submitted or saved

  1. all input are valid

=> allow form to be submitted

when to validate

  1. when form is submitted

=> allow user to enter a valid value before warning him

=> avoid unnecessary warnings but maybe present feedback too late

  1. when a input is losing focus

=> allow user to enter a valid value before warning him

=> very useful for untouched forms

  1. on every keystroke

=> warns a user before he has a chance of entering valid input

=> if applied only on invalid inputs,has the potential of providing more direct feedback

Bonus: Using useReducer()

use-input.js(useReducer版本)
~~
import { useReducer } from “react”;

const initialInputState = {
value: “”,
isTouched: false,
};

const inputStateReducer = (state, action) => {
if (action.type === “INPUT”) {
return { value: action.value, isTouched: state.isTouched };
}
if (action.type === “BLUR”) {
return { isTouched: true, value: state.value };
}
if (action.type === “RESET”) {
return { isTouched: false, value: “” };
}
return inputStateReducer;
};

const useInput = (validateValue) => {
const [inputState, dispatch] = useReducer(
inputStateReducer,
initialInputState
);

const valueIsValid = validateValue(inputState.value);

const hasError = !valueIsValid && inputState.isTouched;

const valueChangeHandler = (event) => {
dispatch({ type: “INPUT”, value: event.target.value });
};

const inputBlurHandler = (event) => {
dispatch({ type: “BLUR” });
};

const reset = () => {
dispatch({ type: “RESET” });
};

return {
value: inputState.value,
hasError,
isValid: valueIsValid,
valueChangeHandler,
inputBlurHandler,
reset,
};
};
export default useInput;
