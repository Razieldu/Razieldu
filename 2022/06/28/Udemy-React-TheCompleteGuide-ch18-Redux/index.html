Another Look At State In React Apps

Redux:state management system for cross-component or app-wide state

一般的 JavaScript 專案也是可以使用 redux

  1. local state

Belongs to a single component.

Should be managed component-interval with useState()/useReducer.

  1. cross-component state

State that affects multiple component.

Required props chain or props drilling.

  1. app-wide state

State that affect all app.(user authentication status).

Required props chain or props drilling.

useContext

Redux vs React Context

  1. Redux 用來管理複雜的 state

  2. 使用 useContext 在過於龐大的 app 會出現龐大的巢狀 provider,把邏輯集中在一個 provider,檔案會過於龐大難以維護,分開處理又會面臨在第一層的 component 出現的大量的巢狀 provider

  3. useContext 適合用來管理變動頻率較低的 state 以及邏輯(登入驗證)

How Redux Works

One Central Data Store(state)

Component set up subscriptions to center data store

Component never directly change the stored data

use Reducer function(not useReducer) to change the store data in the central data store,is a general concept

運作流程

Components dispatch Actions which describe what should be doned,but don’t do it directly,these action are forwarded to the Reducer,the Reducer then do what the Actions want the Reducer to do,then Reducer spits out a new state which will replace the existing state in the Central Data Store,when the state in the data store is updated,subscribing components are notified so that they can update their UI.

Reducer Function:

should be a pure function,same input,same output,no side effect inside of the function

INPUT:

Old state + Dispatched Action

OUTPUT:

New State Object

How To Work With Redux State Correctly

在 reducer 裡面我們 return 的 state object 不會跟既有存在於 central data store 裡面的 state 進行合併而是覆蓋,當我們在 return 的時候必定要把每個 state object 裡面的 state property 都列出來,否則會造成 state 的丟失,always copy and create new object,在更新 state 的時候,always return a brand new state object

Redux Challenges & Introducing Redux Toolkit

  1. dispatch function 裡面的 identifier({type=”INPUT”}),當專案越來越大的時候,由於只要跟 reducer 裡面不一樣,redux 就不會正常運作所以很容易出錯也不好管理

  2. 由於在更新 state 的時候,需要把所有 state property 都列上,當 app 規模越來越大的時候,也會產生不好管理的問題
