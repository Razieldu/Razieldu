Backend Web Development explained

In order to become a full stack web developer,we need to learn about the backend.
Full stack developer= Front-end + Back-end

Back-end: interact with databases or have business logic on a server

A server which will serve up your files ,your HTML,your CSS,your Javascript,a database which can store your user data such as their logs in and passwords,as well as your applications,this is where you create your business logic,you determine your web applications work.舉例來說可能是:計算飛機票價的東西,或是使用支付相關的東西

餐廳客人用餐區: client-side
餐廳廚房:server
儲存食材的倉庫:database

後端程式語言:PHP,Java,Ruby,Nodejs

後端框架:CakePHP,Ruby on rails,Spring,Express. These are things that will speed up development using the particular back end technology you chose. The main job of these frameworks is to simply reduce the amount of repetitive stuff that a developer has to do,and just like how you do not want to write the HTML boilerplate code out every single time you create an HTML page. What the frame work doing is to cut down some of the grunt work and saving you a lot of time.
