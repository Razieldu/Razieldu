Why do we need APIs

A broad definition that a lot of people go by is that APIs is a set of commands,functions,protocols that programmer can use to create software or interact with an external system.

In APIs,some people will consider to be a contract . It is a contract between the data provider and the developer. And essentially what it says is these are all the things that developers can access,and these are the methods,the objects,the protocols that you would use to access them. And we,as the web site that host the data,will try to not change any of these methods or functions without notifying you

In this module,we are going to focus on the type of APIs that allow us to interact with an external system,and most importantly an external server.So,we are going to get hold of some piece of data from a web site and we are going to do that through their API,and we are going to read their documentation to see what are all the things we can do. And we are going to use their API as a menu of things that we can do to interact with their data

我們的伺服器經由API傳送request給someone else的伺服器,再由對方經由API傳送response回來

So,when you want to specify a parameter,remember that you need these symbols in the URL to be able to do that. The parameters come after a question mark ,and then they are set as a key value pair with an equal sign in between. And if you wnat to have more than one parameter ,you separate each of the key value with an ampersand(&)

Through the use of paths and parameters, we are able to narrow down on the data that we get back from the joke API to the precise thing that we want.

What is JSON format

Stands for Javascript Object Notation,the data we get back from Postman looks
remarkably like how we create Javascript objects,where we open up a set of curly braces([]),we have our key ,then our colon(:), then our value ,and each of these key value pairs are then separate by commas(,),and we can embed objects inside other objects

Javascript objects

When we create an actual objects in Javascript we always have a var or a a let keyword in front,we have the name of the object without any quotation marks around it, because it is not a string,and inside our objects,the key are never strings,they are simply written as they are. Now the reason why we use JSON to pass data around on the Internet is because it is in a format that can be readable by human ,but it can also be easily collapsed down to take up as little spaces as possible. So it almost like if your Javascript objects is a full sized wardrobe, but while you are transporting it ,if you bought it from the store and you are taking it home ,you probably do not want to move it as an entire wardrobe.

We can collaspe all of the data down into a single string

As long as we maintain where all the curly braces and the colons and the commas are then at a later stage once we received that JSON as a string, we can build it back up into the original object

Making GET Requests with the Node HTTPS Modules

Client browser is going to make a request to our server and that is going to be a GET request,it is going to try and get the HTML ,CSS and Javascript from our server. At this point,what happens is our server should return all of those piece of data,and that is in the response,but in order to give them the response that includes data from somebody else’s server ,we are going to make a request to that other server,and it is again going to be a GET request ,so that they will give us a response in the form of the data that we need,and we are going to do this via their API,so via the menu that they provided for us to make our requests,where they have specified what are the things that we need to pass over ,like what paremeter,what paths and what key value pairs we have to use in order to get the response and the data we want.So thne, once we get the data ,we can go back and incorporate that data into the files that we send back to our client,the user for our web site.
