React Code is Written in A Declarative way

How is components build?

把HTML,CSS,JS code結合在 components裡面,然後把所有componets組合成一個完整的 user interface,其中以HTML,以及JavaScript的code佔了比較大的比例

React is all about components and about building all these components which we combined together.

React allows you to create re-usable and reactive components consisting of HTML,JS(also CSS).

Declarative approach

React use declarative approach for building these components.

We will always define the desired end state,the target state,or possibly also different target states depending on different conditions and it’s then React’s job to figure out which elements on the actual webpage might need to be added or removed or updated.We don’t write these concrete DOM updating instructions on your own as we would be doing it with just JavaScript,instead with React and React components,we just define the end states and under which condition which state should be used and React will do all of the rest under the hood.

Analyzing a Standard React project

index.js

They are both for the React library.

This create basically the main entry point,the main hook of the overall user interface you are about to build React

It tells the React where this React application, this user interface ,which you build with React in the end should be placed in the web page that is loaded.And that leads us to one other file, the index.html file in the public folder.

This folder is a folder in which we will rarely work but holds one important file, this index.html file.This is the single HTML file,which in the end loaded by the browser here.
This is basically the only HTML file that is being used by the overall application. Because it is so called single page application.It has a single HTML file and all the subsequent changes on the user interface,on the web page will be handled by React,it is also the entry point,the place where the React-driven user interface should be render into so to say.

A div with a id “root” in the index.html,that the div where we want to attach or inject our React-driven user interface,it’s the div where the React user interfaces should be rendered into.

It’s the div with the id “root”,which we select here with some regular JavaScipt code,and then we use a method from the React DOM library to tell React this should be the root of our application,the main place that the React application get rendered to,then we store this root object in a constant,and then on that root object,we can call the render method to tell React what should be rendered in that div that we selected here.

We are in the end telling the ReactDOM that we want to rendering something,the content of this div should be placed with that(root.render(<*App*/>);)
