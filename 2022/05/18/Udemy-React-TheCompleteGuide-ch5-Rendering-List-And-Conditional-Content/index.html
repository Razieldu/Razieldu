understand “keys”

開發人員頁面顯示以下錯誤
“index.js:1 Warning: Each child in a list should have a unique “key” prop.

打開開發人員介面發現新的物件在加入到我們的list的時候,原來最後一樣物件的位置發生變動了,依照我們程式碼規則,添加的新物件應該會顯示在list最前面的位置,但是卻被瀏覽器添加到了最後一項

原因是因為我們沒有告訴React,物件應該要被排列的順序,當我們新增物件的時候,React只是在最後面生成一個div並把所有物件的狀態都更新了

當我們在處理表單資料的時候,都需要幫物件添加類似key的property
