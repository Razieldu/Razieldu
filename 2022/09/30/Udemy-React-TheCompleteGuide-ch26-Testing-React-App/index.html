什麼是測試

  1. 手動測試

寫程式,並預先預覽網頁在瀏覽器上的效果,能看到使用者看到的畫面

很難去涵蓋到所有的情境以及組合狀況

在添加新功能的時候,通常只會測試該功能相關的 component,因此也在無意間種下 bug

  1. 自動測試

我們使用一些程式碼來測試主要部分的程式碼

我們測試我們專案的個別區塊

技術成分很高,但是同時也讓我們能夠一次性測試所有建立的區塊

了解不同的測試種類

  1. Unit Test 單元測試

獨立測試個別的建立區塊程式碼(函數或是部件)

在整個專案當中其實包含了很多的單元測試

假設個別的部分都是可以正常執行並且沒有問題,那整個專案的執行應該也是正常的

  1. Integration Test 整合測試

我們對整合起來的幾個 component 進行測試

數量沒有單元測試那麼多

  1. End to End Test

以一個情境來進行測試,例如:模擬使用者在登入的時候會進行的操作以及流程

數量更少

如何測試

借助工具來進行測試

  1. 我們需要一個工具來進行測試,以及判斷測試成功與否

=> 使用 Jest

  1. 我們需要一個工具能夠模擬或是渲染我們的 React app 或是 component

=> 使用 React Testing Library

但我們使用 create react app 指令來設置專案環境的時候,這兩個功能是已經內建好了

App.test.js

  1. 使用 npm test 指令

假設我們變更了 app.js 的 a 的內容,再執行指令,就會出現測試到錯誤並顯示

  1. 使用指令 npm test 之後變更指定檔案內容,自動測試都會執行,並直接顯示結果

寫測試時會使用到的 3A

Act: 執行需要被測試的邏輯
Assert:把執行結果與期待結果進行比較
