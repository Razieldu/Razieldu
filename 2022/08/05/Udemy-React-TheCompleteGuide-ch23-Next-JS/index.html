  1. File-based routing
  2. Page-Pre Re-rendering
  3. Data-Fetching
  4. Adding an API

What is NextJS?

  1. A Fullstack framework for React JS
  2. Next JS makes building React apps easier

Key Feature 1: Built-in Server-side Rendering (Improved SEO!)

Server side rendering(SSR):preparing the content of a server side instead of client side

  1. 優化 SEO:client side render 導致我們的 app 在 html 的部分是一片空白不利於搜尋引擎找到我們的 app,使用 NextJS 可以改善這個問題

  2. 因為是 client side render,所以使用者在使用的時候會看到讀取個過程,因為某些資料需要先進行獲取再進行渲染

  3. 結合 client-side and server-side,在 server 獲取資料,然後渲染完成頁面

Key Feature 2: Simplified Routing with File-based Routing

使用資料夾結構以及檔案名稱來當分頁面的 url

Key Feature 3: Build Fullstack Apps

We can easily add backend(server-side) code to our app

Creating a New Next.js Project & App

在 page 資料夾建立檔案

  1. index.js:網址=> our domain.com/
  2. news.js :網址=> our domain.com/news

使用檔案名稱作為網址

index.js

news.js

Adding Nested Paths & Pages (Nested Routes)

  1. index.js 網址:localhost3000 ///使用 index.js 來定義母網頁
  2. /news/index.js 網址:localhost3000/news
  3. /news/newspage2.js 網址:localhost3000/news/newspage2

Creating Dynamic Pages (with Parameters)

網址變為:localhost3000/news/任何內容

Extracting Dynamic Parameter Values

可以使用取得的 newId 向後端發出請求

Linking Between Pages

檔案之間的連結使用 Link 來處理

建立專案結構

以檔名為頁面的 url,一般的 react 組件擺放在 component 資料夾裡面,只有在 page 資料夾裡面的頁面才會被 Next JS 生成頁面,components 資料夾可以擺放一般的 react code,進行取用

_app.js 檔案(根檔案)

我們把 navigation 組件在這邊導入並且進行包覆,如此一來我們就不需要在每一個需要用到的 component 導入,只需要導入一次,其他的頁面都會有 navigaton bar

Using Programmatic (Imperative) Navigation

使用自定義function搭配useRouter,以及從props取得的id來進行動態頁面的生成
