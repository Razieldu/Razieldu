z-index相關在第五張CSS

使tindog專案頁面的手機圖案顯示在section feature之後

根據HTML的排列規則,features寫在iphone6的之後,因排序比較前面,所以也不需要再設定z-index

css media 設定

因為頁面大小變換的時候,圖片會覆蓋到文字,且文字隨著頁面變小會縮成一塊

把位置設定成static讓排序再依照原來的次序,就不會產生覆蓋的問題,rotate的值是以絕對值來定義,
rotata(360)=rotate(0),同時在section title 的部分設定置中,解決文字擠成一塊的問題

Code refactoring

讓編碼更整齊更容易閱讀,出現問題時也能直接知道是個部分出錯

You should refactor your code and maintain it to keep your code in a state as if
you expect the next person who is going to have to maintain your code to be a
axe murderer.

  1. Readability
  2. Modularity
  3. Efficiency
  4. Length

把範圍廣的定義selector先進行整理,觀察有沒有能夠獨立出來的部分,都獨立建一個class再進行定義

這也是一個表示方式

text-align:center,出現很多次,直接在body定義
padding通常會用在container之類的selector,盡量使用一樣的padding,以便格式上的一致
body部分的一開始,使用了padding為 3% 15% 7%的container-fluid,因為整個程式碼出現很多次
padding:7% 15%,所以索性把container-fluid的padding改成7% 15%(上端的3趴程式碼梳理完成再設定),有一個section需要padding 7% 15%,直接在section裡最上方添加一個container-fluid
section:cta 以及section:footer都需要7% 15%,因此都添加一個div class=container-fluid

在需要padding:7% 15%且已經有定義class的部分之最後添加container-fluid

代表在這個部分的container-fluid會變為3%15%7%
