HTML divs

Stands for :Content division element
Div: is a special HTML element that allows us to divide our content up on our website
so that we can structure each dic seperatelydon’t do anything unless you use CSS with it

  1. 點擊開發人員選項可以看到各個元素的資料,進行暫時性的修改,確定了之後再到應用程式裡面進行修正

Box model of website design

padding:物件content到border之間的距離
margin:物件的border與其他物件的border的距離

CSS display property( 4 types)

1.block elements

  1. 即使位於頁面的最左邊,剩下的空間從左到右都會被同一個元素佔據

2.Inline elements

  1. span 如此的彈性為什麼不用span來定義物件,因為無法更改span的寬度
  2. 使用display進行物件屬性的轉換
  3. 把block element display成inline element,但是本來可以更改的width變成無法更改
  4. 把inline element display成block element, 可以更改width但是也具有block element的特性,佔據同列的剩餘空間

3.Inline-block element

能夠更改高以及寬,同時也不會佔據剩餘的空間

4.None

  1. 使用display 轉換成此屬性,會讓整個物件直接消失,此物件種類可以用來隱藏一些資訊
  2. 使用visibility: 時,會像隱形一樣但是在網頁仍然會依照排列順序出現在該在地方,只是
    是隱形起來的

CSS POSITIONING

1.HTML position rule

  1. Content is everything:inline elements only take as much width and as much height as
    the content,content(物件內容) 高度以及寬度的因素是決定物件位置的第一個因素

  2. order comes from code :程式碼呈現的順序

  3. children sit on parents: X and Y 決定物件位置,同時也有一個Z決定物件的呈現(靠近viewer)先後,假設有個div裡面有個h1,如果dic的順序在h1之前,則我們將無法看到h1

By default our HTML elements are children ,they will sit on top of the parents element

最上層:span,第二層:h1,第三層:div

2.CSS POSITION

  1. Static position: 依照HTML的原來的位置規則

  2. Relative position:相較原來的位置(static position)的移動量

Coordinates:TOP,BOTTOM,LEFT,RIGHT
TOP:位移量將在物件上方,並且以TOP margin為基準,其餘的以此類推

不會影響任何其他的物件的位置,或許會有覆蓋的問題

就像留下一個透明的空殼在原本的位子,去讓其他物件維持位置,然後原物件已經做了位移

  1. Absolute position

relative to his parents

以parent(div)的右邊margin為原本位置,以物件右邊margin為基準,位移量在右為30px

In most cases,the parent is simply the entire body of the page

跟relative position在原本位子留下空殼的概念不一樣,absolute position是跳脫HTML規則,直接把整個物件位移過去,剩下的物件會依照HTML規則排列

賦予absolute position會使物件 take any HTML element out of the flow of the document,and
is no longer following and stacking the natural orders

  1. Fixed position

    不會隨著頁面的上下左右移動而移動,會固定在原位,但其他的物件會變動

CSS Z-index

Whatever comes first in he HTML code, so closer to the top of the file,will
be closer to the back,and as you go down in the HTML file,we come further and
further forwards towards ourselves and away from screen

z-index only works if your elements are positioned,so it has a positioning.
