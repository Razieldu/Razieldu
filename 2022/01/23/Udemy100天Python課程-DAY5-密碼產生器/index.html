密碼產生器

題目

導師的解法1 (簡易版本) - Order not randomised:

重點筆記

random.choice(seq), choose a random item from a sequence. seq can be a list, tuple(元組), string,or any iterable like range.

導師的解法1 (困難版本) - Order randomised:

Append and Extend

  1. random.shuffle(list1) 將序列的所有元素隨機排列
  • append()
  • list1=[1,2,3],list1.append(7),list1=[1,2,3,7]
  • list1=[1,2,3],list1.append[7,8],list1=[1,2,3,[ 7,8 ]]
  • 添加對象可以是元素或是串列或是dictionary
  • 添加對象會以原來的資料形式加入
  • extend()
  • list1=[1,2,3],list1.extend(7)
  • 出現error,因為只能是串列
  • list1=[1,2,3],list1.extend[7,8]
  • list1=[1,2,3,7,8]

個人解法 (簡單版本) - Order not randomised:

思路

  1. 三個串列個別計算出來後,再進行相加
  2. total_letter=””的假設要在forloop之前

個人解法 (困難版本) - Order not randomised:

思路

  1. 使用for num in new 迴圈會執行到產生三個亂數就停止,因為串列的元素個數在減少,導致迴圈次數也減少,因此另外假設一個new1的串列,設為for num in new1
  2. 將new_new儲存在new_new1之後再進行remove,因為如果不多假設一個new_new1,只有一個new_new,在進行remove的時候會發生串列裡面沒有元素可以扣除的錯誤(2儲存到new_new,remove掉2,產生3,儲存值變為23,remove掉23,但串列已經沒有2這個元素,因此會發生錯誤)
  3. 串列.remove(),只能套用在list的對象,因此使用list()把new1以及new兩個string都轉換成list
