What is REST

REST:REpresentational State TRansfer

Client make a request to the Server on the Internet this is done through a HTTP request(Hypertext Transfer Protocol request)

or a FTP resquest(File Transfer Protocol request),if you use the wrong language,you might not get the result you are looking for.

HTTPS request:stands for each HTTP secure request that your requests and responses are going across the Internet and can be potentially tapped in by lots of people,so if you want to make it secure so you do not want to be on a website entering your request details and it is just going through a HTTP request because somebody might hijack that is not very secure,so we can use cryptography and encrypt our request,this is kind of having a secret code language.

The client has successfully made the request to the server and the server is able to serve up, so it is a valid request instead of a 404,then the server should respond with the result of what it is that you requested.

In order to give us that result,the server might do one of two things.It might do some computation,run some code to work out the result that we need.If our server was a simple calculator web application,or the server might need to communicate with our database in order to grab the relevent pieces of data that the client requested.

The client will talk to the server using an HTTP request, a specific language that the server can understand. The server will have a whole bunch of APIs which are services that it can expose for client to be able to tap into.

We are going to build an API,we are kind of like building the menu of things that our server can respond to when a client makes a request.

What does it mean to make our API RESTful?

REST is essentially an architectural style to design APIs and it is /definitely not the only architectural style(others:SOAP,GraphQL,FALCOR)

Two importany part to make your API RESTful

  1. Use HTTP Request Verbs

  2. Use Specific Pattern of Routes/Endpoints URLs

What is exactly the HTTP verbs

  1. GET:is basically that same as read
  2. POST:correspond to the create word in our CRUD
  3. PUT,PATCH:both go into our database and update some piece of data

PUT:You are updating your database by sending an entire entry to replace the previous one

PATCH:You only sending the piece of data that needs to be updated

  1. DELETE:delete a particular piece of data in our database

Specific Pattern of Routes/Endpoints URLs

In our server,we can specify specific routes or URLs in order to access certain resources

類似先前使用app.get(“/“+特定文字),可以有不同的頁面呈現

The client can be a number of things,it could be a browser that is trying to load up the pages in which case it will make a GET request to our server via our API and our server will send over the relevant HTML,CSS,Javascript files to be able to render that web page in the browser
