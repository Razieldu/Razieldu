How to Deploy Web Apps with a Database

In order to access our website, we go to localhost:3000 and that means that we are using our local computer as the server to serve up the information and data in our data and our website.And that backend relies on Node.js. Now in the last module,we also learned how to persist our data to save it to a database using MongoDB. The MongoDB database is also served up from our local computer and the URLs that we use to access be it our server or MongoDB all have the word localhost in it.

We have been accessing app, we have been using the hypertext transfer protocol(http).We have been going to a port 3000 on the localhost and we needed to access our database,we have been accessing it at port 27017 also on our localhost.Now the part before the colon differs because here we are using(http:localhost:3000) the hypertext transfer protocol,so a set of rules that govern how we transfer data across the Internet to access web pages.But here(mongodb://localhost:27017) we are using the MongoDB rules which determines how you can access data in MongoDB database.

  1. 先前是把專案放上heroku(但是僅限於沒有資料庫的app)
  2. 因為資料庫在我們本地的電腦,假設把app.js的程式碼放上heroku,我們也無法隨時都能打開本地的資料庫,因此我們需要一個24小時都能用的伺服器並把資料庫放上去

MongoDB Atlas

MongoDB’S own cloud service,this is going to allow us to have a web based address. MongoDB Atlas is going to host our database and served up the data whenever it is needed.So,that means that our Node.js app on Heroku wiil be able to make the necessary requests to our database that is hosted on the MongoDB Atlas cloud service and we will be able to delete,update,or to get data back from that database and that will all happen behind the scenes.Any users who needs to access your website will be going through your app.herokuapp.com and then your backend will liaise(取得聯絡) with MongoDB Atlas in order to talk to the database,so we no longer have any need for hosting our server or our database locally
